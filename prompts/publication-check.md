# Publication check

Publication layer. Final pass before a report travels. Remove unsupported
intent, preserve sourced structure.

## Checklist

- [ ] **Evidence stays inside the claim.** Every structural statement cites a
      source and an evidence class. No claim is upgraded beyond its source.
- [ ] **Role vs intent vs wrongdoing.** A sourced role may be named. Intent
      claims require intent evidence. Wrongdoing claims require wrongdoing
      evidence. Do not let a sourced role smuggle an unsourced intent.
- [ ] **No dropped nodes (deference trap).** A sourced public node is not
      removed because naming it feels accusatory.
- [ ] **No carried intent (innuendo trap).** A named, sourced node does not
      carry an intent the ledger does not support.
- [ ] **Scope held (scope trap).** No private individual has been
      neutral-retrieved into a dossier. Public capacities only.
- [ ] **Falsification line present.** Every verdict states what would clear it,
      even when you suspect it cannot be produced.
- [ ] **Contamination as a bucket with reasons.** No bare numeric score travels.
- [ ] **Sourcing gate respected.** If fewer than three load-bearing fields are
      sourced, the output is object type, route, known evidence, missing
      fields, and a neutral pull-list — not a verdict.

## Standing line

> Output is a structural assessment, not an allegation of wrongdoing.

If any box is unchecked, the report is not finished.
